# プレイヤー戦歴ビュー（シーズン別） — 要件定義

> ステータス: **要件定義（実装前）**。コード実装は未着手。
> 概要: 麻雀リーグの順位画面でユーザーをタップすると、**そのシーズンの戦歴**を閲覧できるようにする。シーズンを切り替えると、そのシーズンの順位表・戦歴に切り替わる。

---

## 1. 背景・目的
- リーグの順位リストから、各プレイヤーの**戦歴（各半荘の着順・持ち点）**と**通算成績**を見たい。
- 閲覧は**シーズン単位**。2026年シーズンを開けば2026年の戦歴、2027年シーズンを開けば2027年の戦歴。
- ゲスト開放（`docs/ゲスト利用者-ゲーム開放-要件定義.md`）とは独立だが、**ゲストも順位表に載る/閲覧できる**点で整合させる。

## 2. 現状確認（実現性）
- `MahjongStanding`（順位表）は既に **`lineUserId` を含む**＝タップ用の識別子がクライアントに既にある。
- 集計値は計算済み: `gamesPlayed`（戦数）/ `average`（AVG）/ `firstRate`（1位率）/ `top2Rate`（連対率）/ `firstCount` / `top2Count` / `csEligible` / `rank` / `tier`。→ **戦歴ヘッダーの集計はそのまま流用**。
- 各半荘の生データは `mahjongTables`（`members[].points/rank/reportedAt`・`eventDate`・`round`・`status`・`seasonId`）。→ **1試合ごとの戦歴とAVG推移を再構成可能**。
- `standings` API は既に **`?seasonId=` 指定対応**（未指定はアクティブ）。
- ❗ **portal にシーズン切替UIは無い**／**portal向けシーズン一覧APIも無い**（管理側 `/api/admin/scoreboard/seasons` のみ）→ 本機能で**新規に用意**する。

---

## 3. スコープ（本機能に含むもの）
本機能は次の2つで構成する。

### A. シーズン選択（前提・同梱）
- portal リーグ画面（`MahjongLeagueView` の「リーグ」タブ）に**シーズン選択**を追加。
- 選択シーズンで**順位表（standings）を表示**（standings は既に `?seasonId=` 対応）。
- 既定の選択は**アクティブシーズン**。過去シーズンも選べる。
- 「参加」「申告」タブは**アクティブシーズン固定**（過去シーズンへの参加表明/申告は不可）。シーズン選択は**リーグ（順位/戦歴）閲覧にのみ効く**。

### B. プレイヤー戦歴ビュー
- リーグタブの**順位リスト**（＋任意でピラミッドのアバター）でユーザーをタップ → **戦歴ボトムシート**（選択中シーズン分）。
- 表示内容:
  - **ヘッダー**: アバター / 氏名 / tier（M1/M2/M3）/ そのシーズンの順位
  - **集計**（standings流用）: 戦数・AVG・1位率・連対率・CS資格
  - **戦歴リスト**: 各半荘の「日付 / 第n回戦 / 着順 / 持ち点」（`status=completed` のみ・新しい順）
  - **AVG推移スパークライン（今回含める）**: そのシーズンの試合を時系列に並べ、**各試合終了時点の累積アベレージ**を折れ線表示。

---

## 4. API（新規）
| API | 認可 | 返却 |
|-----|------|------|
| `GET /api/mahjong/seasons` | `requireGameUser` | `[{ seasonId, name, year, status, isActive }]`（gameCategory=mahjong のシーズン一覧。新しい順） |
| `GET /api/mahjong/players/[lineUserId]/history?seasonId=` | `requireGameUser` | `{ player:{displayName,pictureUrl,tier,rank}, standing:{gamesPlayed,average,firstRate,top2Rate,csEligible}, games:[{eventDate,round,points,rank,tableId}], avgTrend:[{date, cumulativeAverage}] }` |

- **戦歴データ源**: `mahjongTables` を `seasonId` ＋ `memberIds array-contains lineUserId` ＋ `status=completed` で取得し、その人の `{points, rank}` を抽出。`eventDate`→`round` 昇順で並べ、`avgTrend` は累積平均を逐次計算。
- **集計の整合**: `standing` は既存 `computeStandings(seasonId)` の該当ユーザー値と一致させる（再計算 or 流用）。
- 認可は **`requireGameUser`**（会員・ゲスト両方が、誰の戦歴も閲覧可＝リーグの公開競技データ）。← ゲスト要件と整合。

---

## 5. UI/UX
- **シーズン選択**: リーグタブ上部にセレクタ（ドロップダウン or 年タブ）。切替で順位表が再取得。
- **タップ導線**: 順位リストの行タップで戦歴シート（既存のボトムシート様式 `src/components/ui/Sheet.tsx` 等に合わせる）。
- **戦歴シート**: ヘッダー集計 → AVG推移スパークライン → 戦歴リスト の縦並び。
- **空状態**: そのシーズンに試合が無いユーザーは「戦歴なし」。
- アクセント/配色は既存の麻雀UI（TILES案・M1/M2/M3色）に合わせる。

---

## 6. データモデル
- 既存 `mahjongTables` / `MahjongStanding` のみで実現可能（**スキーマ追加なし**）。
- ゲストも `lineUserId` 起点で同様に戦歴を持つ（ゲスト要件と整合）。

---

## 7. ゲスト/プレビュー整合
- ゲストも順位表に載るため、**戦歴閲覧の対象**かつ**閲覧者にもなれる**（`requireGameUser`）。
- プレビュー（ダミー）: ダミーの `seasons` 一覧と、プレイヤー戦歴（既存 `dummyTables` を流用し1〜数試合分）を用意すれば、preview で確認可能。

---

## 8. プライバシー
- `standings` が既に `lineUserId` を返しているため、それをキーに戦歴を引く（**新たな個人情報露出は増えない**）。
- 将来 `lineUserId` を隠したい場合は不透明な playerId に差し替える余地を残す（本機能では既存踏襲）。

---

## 9. 未決事項
- シーズン選択UIの形（ドロップダウン / 年タブ）。
- スパークラインの指標を「累積AVG」で確定（着順推移や持ち点推移は出さない、で良いか）。
- タップ対象を順位リストのみにするか、ピラミッドのアバターも含めるか。
- 過去シーズンの「tier/順位」をそのシーズンの確定値で出す（再計算の基準）。

## 10. 実装ブレークダウン（参考・本書では実装しない）
1. **シーズン一覧API** `GET /api/mahjong/seasons`（portal用・requireGameUser）。
2. **シーズン選択UI**: リーグタブにセレクタ追加、standings を選択 seasonId で取得。
3. **戦歴API** `GET /api/mahjong/players/[lineUserId]/history`（mahjongTables 集計＋avgTrend）。
4. **戦歴シートUI**: ヘッダー集計＋AVG推移スパークライン＋戦歴リスト。順位リストのタップ配線。
5. **ダミー**: `seasons` と プレイヤー戦歴のダミーを追加（preview確認用）。
6. **テスト**: history 集計（着順/AVG推移）と認可（requireGameUser）のユニットテスト。

> 関連: 認可方針は `docs/ゲスト利用者-ゲーム開放-要件定義.md`、麻雀の現行仕様は `CLAUDE.md` を参照。
